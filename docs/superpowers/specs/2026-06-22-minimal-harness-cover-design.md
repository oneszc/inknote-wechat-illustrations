# 最小 Harness 文章封面设计

## 目标

为《让 Codex 不再像刚入职：我的最小 Harness 搭建方法》生成一张微信公众号封面，让读者一眼理解：用最少的 5 个项目文件，为 Codex 准备一个能持续工作的项目环境。

## 构图

- 比例：2.35:1，核心内容保持居中，适应公众号预览裁切。
- 主体：一个小型 Codex 机器人站在由 5 个文件卡片组成的工作台前。
- 文件卡片：`AGENTS`、`CONTEXT`、`STATE`、`TASKS`、`VERIFY`，用短英文避免画面拥挤。
- 标题：只保留「最小 Harness」，使用粗黑马克笔手写字。
- 辅助信息：用一条小箭头表现 Codex 从“刚入职”走向“开始工作”，不添加长句。

## 视觉风格

- 白色横线笔记本纸、浅蓝横线和一条红色页边线。
- 粗黑马克笔线稿，方框和箭头保留手绘的不规则感。
- 仅使用一处红色强调，例如标题下划线或完成勾。
- 保持留白和手机端可读性，避免精致矢量图、数字字体、3D、渐变和复杂信息图。

## 验收标准

- 输出为单张 2.35:1 横向封面。
- 标题「最小 Harness」清楚、准确、像手写而非印刷字体。
- 5 个文件卡片可辨认，但不抢夺主标题。
- 画面只表达一个核心概念：5 个文件组成 Codex 的最小工作台。
- 视觉风格与当前 `assets/examples/` 保持一致。
